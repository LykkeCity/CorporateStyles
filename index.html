< p > - In principle, it would be possible to further simplify the trading client, by only letting it communicate with the exchange and not connecting to the distributed ledger. This slightly reduces the risk of attacks, as there is one less communication channel to the outside. But it the trader would have to rely completely on the data it gets from the exchange and would not be able to independently verify transactions any more.


< q > - The current system where every user is a network node is not the intended configuration for large scale. That would be like every Usenet user runs their own NNTP server. The design supports letting users just be users.
